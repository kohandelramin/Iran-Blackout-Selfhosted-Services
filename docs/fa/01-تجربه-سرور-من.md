# تجربه سرور من

این ریپو را مثل یادداشت‌های شخصی خودم نوشتم. هدفم این بود چند سرویس ساده داشته باشم که وقتی اپ‌های معمولی کند، فیلتر، یا ناپایدار شدند، حداقل ارتباط پایه از بین نرود.

قرار نیست کل اینترنت را جایگزین کند. فقط می‌خواهم این‌ها زنده بمانند:

- جلسه آنلاین با Jitsi
- چت صوتی با TeamSpeak 3
- چت روم با Mattermost
- استریم ساده با Owncast
- فایل شیرینگ با FileBrowser

قسمت سخت همیشه خود سرویس‌ها نبود. بیشتر دردسرها اطرافشان بود: Docker image دانلود نمی‌شد، DNS درست جواب نمی‌داد، SSL و CDN redirect مشکل می‌ساخت، پورت UDP فراموش می‌شد، یا کاربران از ISPهای مختلف تجربه متفاوتی داشتند.

## چیزی که برای من بهتر جواب داد

- همه چیز Docker based باشد.
- هر سرویس پوشه خودش را داشته باشد.
- Nginx reverse proxy تا جای ممکن روی host باشد.
- UFW rule ها واضح باشند.
- دامنه `.ir` و ساب‌دامین‌ها از قبل آماده باشند.
- تنظیمات mirror ها محلی ذخیره شده باشند.
- بکاپ `.env` بیرون از Git نگهداری شود.
